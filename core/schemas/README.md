# Canonical schemas

JSON Schema documents that form the **public contract** of AGTP. Every
language library (`agtp-python`, `agtp-php`, `agtp-go`, ...) loads
these schemas and validates the shapes they exchange against them.
A drift CI test (in `tests/schemas/`) fails the build when the Python
dataclasses no longer match the frozen schemas in this directory.

## Public vs internal

Two categories of dataclass exist in this repository. Only the public
ones live here.

### Public (in this directory)

These shapes cross the gateway socket, ride on the AGTP wire, or are
served as documents to peer agents. A breaking change to any of them
is a versioned, contract-level event.

| Shape | Source dataclass | What it is |
|---|---|---|
| `endpoint-context.schema.json` | `agtp.handlers.EndpointContext` | The parsed request envelope handed to handler code. |
| `endpoint-response.schema.json` | `agtp.handlers.EndpointResponse` | The success-shape returned from a handler. |
| `endpoint-error.schema.json` | `agtp.handlers.EndpointError` | The declared-failure shape returned from a handler. |
| `agent-document.schema.json` | `core.identity.AgentDocument` (v2) | The per-agent identity document served by DESCRIBE. |
| `server-manifest.schema.json` | `core.manifest.ServerManifest` | The server-level manifest served by DISCOVER. |
| `gateway-handshake.schema.json` | (no dataclass — gateway-only) | The `hello` / `welcome` / `register` / `request` / `response` frames defined in [`gateway-protocol-v1.md`](../../docs/architecture/gateway-protocol-v1.md). |

### Internal (NOT in this directory; subject to change)

These shapes are implementation details of the daemon. They are
allowed to change between releases without a contract-level version
bump. Do not stabilize them by moving them into this directory.

- `core.endpoint.SemanticBlock` — endpoint metadata, consumed only by
  the manifest renderer and the synthesis runtime.
- `core.endpoint.ParamSpec` / `FieldSpec` — schema authoring helper;
  the on-wire output is plain JSON Schema.
- `core.endpoint.HandlerBinding` — operator-config artifact, never
  crosses the gateway.
- `core.endpoint.EndpointSpec` — operator-config artifact. The gateway
  carries a flattened subset (method, path, schemas, errors, scopes);
  the rest is implementation.
- `core.endpoint.EndpointDeprecation` — operator-config artifact,
  rendered into the manifest but not its own shape.
- Every `core.manifest.*` sub-block other than `ServerManifest`
  itself (`ServerInfoBlock`, `PolicyBlock`, `APIEndpoint`,
  `HostedProtocol`) — these compose into `ServerManifest`, which
  freezes their public shape transitively. They have no independent
  schema in this directory.

When a contributor wants to promote an internal type to the public
contract, it gets its own schema file here and its own test in
`tests/schemas/`. That promotion is a deliberate review event, not
an accident.

## Drift detection

`tests/schemas/test_dataclass_drift.py` walks the public dataclasses,
projects them into JSON Schema, and diffs against the frozen files
in this directory. The test fails on:

- A new required field that didn't exist in the schema.
- A removed field that the schema still requires.
- A type change on an existing field.
- A new value in an enum that the schema doesn't admit.

It does not fail on additions of optional fields with backward-
compatible defaults; those are accretive and safe.

## Versioning

Each schema carries a `$id` and a `version` in its `info` block.
v1 schemas are frozen for the lifetime of gateway protocol v1
(see [`gateway-protocol-v1.md` §12.1](../../docs/architecture/gateway-protocol-v1.md#121-when-v2-cuts)).
Additive optional fields land via minor bumps inside v1. Breaking
changes wait for v2 — and v2 requires both the mTLS gate and the
new-frame-shape gate to fire.

## What this directory is not

- Not a JSON Schema **dialect** specification. The schemas use
  Draft 2020-12; refer to that specification for keywords.
- Not the AGTP method catalog. That lives in
  [`../methods.json`](../methods.json) and is generated by
  `scripts/build_methods.py`.
- Not a configuration format. Operator config (endpoint TOML,
  server TOML) is documented under [`../../docs/`](../../docs/).
